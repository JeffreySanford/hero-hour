# Sprint 2.3 — Reflection, Review, and Recommendation Seeds

## Feature Focus

- Daily reflection
- Weekly review
- Blocker capture
- Review summaries

## User Stories

- As a user, I want a short daily reflection so that I can inspect how the day went.
- As a user, I want a weekly review so that I can see patterns and adjust.
- As a user, I want simple improvement suggestions based on my week so that the app helps me improve.
- As a user, I want the review tone to be supportive rather than judgmental so that I will keep using it.

## Technical Tasks

- Create DailyReflection and WeeklyReviewSnapshot entities
- Build daily reflection UI and weekly review screen
- Aggregate planned vs completed, balance, streak, and blockers
- Add first deterministic review-summary recommendation rules
- Add privacy boundaries for reflection data

## Acceptance Criteria

- Daily reflections can be saved and retrieved
- Weekly review is based on actual data
- Recommendation summary contains rationale

## Estimates / Priority / Dependencies

- Priority: High
- Story Points: 28
- Dependencies: Sprint 2.1 and 2.2 telemetry
