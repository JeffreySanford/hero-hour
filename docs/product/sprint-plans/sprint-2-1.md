# Sprint 2.1 — Daily Planning, Estimation, and Sub-Quests

## Feature Focus

- Daily planning
- Effort scoring
- Overload visibility
- Parent-child quest structure

## User Stories

- As a user, I want to create daily quests so that I can turn my day into a manageable plan.
- As a user, I want to estimate quest effort so that I think before committing.
- As a user, I want the app to warn me when I overload a day so that I can adjust early.
- As a user, I want to break a large quest into sub-quests so that it becomes approachable.
- As a user, I want to reorder quests by priority so that my plan reflects reality.

## Technical Tasks

- Build daily planning screen and quest editor
- Create DailyQuest, SubQuest, and estimate fields
- Build total effort and overload scoring service
- Add drag/reorder persistence
- Add quest template/recent quest suggestion support
- Add planning telemetry events

## Acceptance Criteria

- Users can create, edit, reorder, estimate, and decompose daily quests
- Overloaded days are surfaced clearly
- Parent and sub-quest progress is visible

## Estimates / Priority / Dependencies

- Priority: High
- Story Points: 36
- Dependencies: PI 1 complete
