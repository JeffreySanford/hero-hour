# Sprint 3.2 — Avatar Progression, Companion, and Achievement Depth

## Feature Focus

- Identity growth
- Emotional attachment
- Achievement system

## User Stories

- As a user, I want my avatar to evolve over time so that my progress feels personal.
- As a user, I want a companion that responds to my consistency so that the world feels alive.
- As a user, I want achievements for planning, balance, and consistency so that good habits are celebrated.
- As a user, I want an achievement gallery so that progress accumulates visibly.

## Technical Tasks

- Create CompanionState, AchievementDefinition, AchievementProgress, and cosmetic unlock entities
- Build avatar progression and cosmetic unlock UI
- Build companion behavior states and event-driven transitions
- Build achievement evaluation engine and gallery
- Emit achievement and companion telemetry

## Acceptance Criteria

- Avatar progression and companion state persist correctly
- Achievement system supports progress-based and instant achievements
- Unlocks are idempotent and explainable

## Estimates / Priority / Dependencies

- Priority: Medium-High
- Story Points: 34
- Dependencies: Sprint 3.1 progression scaffolding
