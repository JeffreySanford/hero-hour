# PI 2 – SAFe-Style Features

## Epic: Habit Formation and Planning Intelligence Foundations

### Feature 2.1 – Daily Quest Planning

Benefit Hypothesis: If users can plan their day through quests, they will start adopting intentional time structure rather than reactive logging.
Acceptance Criteria:

- daily quests can be created and reordered
- main vs optional quests are supported
- planning view reflects real state
Stories:
- As a user, I want to create daily quests so that my day has structure.
- As a user, I want to reorder quests so that I can prioritize.
- As a user, I want optional quests so that I can separate must-do from nice-to-have work.

### Feature 2.2 – Effort Estimation and Load Awareness

Benefit Hypothesis: If users estimate effort before work and see overload warnings, they will begin building realistic planning habits.
Acceptance Criteria:

- effort values can be assigned
- daily load summary exists
- overloaded days are visibly flagged
Stories:
- As a user, I want to estimate quest effort so that I think before committing.
- As a user, I want to see total planned load so that I do not overload the day.
- As a user, I want warnings on unrealistic plans so that I can adjust early.

### Feature 2.3 – Quest Decomposition

Benefit Hypothesis: If users can break large work into sub-quests, completion rates and momentum will improve.
Acceptance Criteria:

- parent/sub-quest relationships work
- partial progress is visible
- decomposition can be encouraged by the system
Stories:
- As a user, I want to split big quests into smaller parts so that work feels manageable.
- As a user, I want sub-quests to roll up into the parent quest so that progress is visible.

### Feature 2.4 – Streaks and Consistency Systems

Benefit Hypothesis: If consistency is rewarded visibly, users will return more frequently and form habits.
Acceptance Criteria:

- at least one daily streak and one focus streak exist
- streak rules are visible and fair
- streak visuals are prominent but not oppressive
Stories:
- As a user, I want a streak for showing up so that return behavior is rewarded.
- As a user, I want focus streaks so that concentration matters too.
- As a user, I want clear streak rules so that the system feels fair.

### Feature 2.5 – Focus Sessions and Sustainable Pace

Benefit Hypothesis: If users are guided into focused work blocks and breaks, the app will reinforce healthier execution rhythms.
Acceptance Criteria:

- guided focus sessions can be started
- uninterrupted completion can earn bonus rewards
- break/recovery prompts are available
Stories:
- As a user, I want guided focus sessions so that I can work in structured blocks.
- As a user, I want bonus rewards for uninterrupted focus so that concentration feels valuable.
- As a user, I want break prompts so that the app supports sustainable pace.

### Feature 2.6 – Reflection and Weekly Review

Benefit Hypothesis: If users reflect daily and review weekly patterns, they will start adapting behavior rather than just recording it.
Acceptance Criteria:

- daily reflections can be submitted
- weekly review summarizes actual patterns
- at least one recommendation appears in review
Stories:
- As a user, I want a short end-of-day reflection so that I can inspect how the day went.
- As a user, I want a weekly review so that I can see what patterns are emerging.
- As a user, I want the review to suggest improvements so that I can adapt.

### Feature 2.7 – Achievement Foundations and Quest History

Benefit Hypothesis: If the app preserves and celebrates visible accomplishment history, users will feel that time invested is cumulative and meaningful.
Acceptance Criteria:

- achievements can be earned and viewed
- recent quest history is visible
- notes and session context can be retained
Stories:
- As a user, I want achievements for good planning and consistency so that smart behavior is rewarded.
- As a user, I want to review completed quests so that I can remember what I accomplished.
- As a user, I want notes attached to sessions so that later reviews are richer.
