# Privacy Principles

- Collect only data needed for personalization and coaching
- Explain why each data category is collected
- Separate auth identity from life-profile and gameplay domains
- Private strategy profile by default
- No default sale or sharing of user-linked profiling data
- Moderation access is scoped and limited
- Admin access to sensitive reads should be auditable
- Explain life-profile data use during onboarding
- Separate optional reminder consent from mandatory terms/privacy acceptance
- Require explicit opt-in for any future data sharing or partner use
- Support profile editing and deletion flows
- Document data classification, retention, deletion, and audit rules
- Privacy-sensitive areas (life roles, frictions, habit anchors, reflections, strategy profile, inferred patterns) must have explicit read rules
